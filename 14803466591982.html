Java中的构造代码块和静态代码块

2016/11/28 23:24 下午 posted in  Java

构造代码块

构造代码块，就是类当中直接包含的不带任何修饰的代码块。例如：

class Foo {
    private String name;
    {
        System.out.println("我是构造代码块");
    }
    Foo() {
        System.out.println("无参构造方法");
    }
    Foo(String name) {
        System.out.println("有参构造方法");
    }
}

public class Bar {
    public static void main(String[] args) {
        Foo f1 = new Foo();
        Foo f2 = new Foo("liubang");
    }
}

编译执行后的结果为：

liubang@venux:~ $ java Bar
我是构造代码块
无参构造方法
我是构造代码块
有参构造方法

我们可以发现，无论使用哪个构造方法，都会先执行构造代码块。因此我们可以总结出以下特点：
1. 构造代码块是被类体直接包含的；
2. 构造代码块是随着类的实例化而执行；
3. 构造代码块可以给所有对象初始化；
4. 构造代码块先于构造方法执行；

静态代码块

静态代码块就是被static修饰的代码块。例如：

class Foo {
    static {
        System.out.println("A");
    }
    Foo() {
        System.out.println("construct run");
    }
    static {
        System.out.println("B");
    }
    
    public static void say() {
        System.out.println("say run");
    }
}

public class Bar {
    static {
        System.out.println("C");
    }
    public static void main(String[] args) {
        Foo.say();
        Foo f1 = new Foo();
        System.out.println("main run");
    } 
    static {
        System.out.println("D");
    }
}

编译运行的结果为：

liubang@venux:~ $ java Bar
C
D
A
B
say run
construct run
main run

从上面的结果中我们不难发现，静态代码块具有以下特点：
1. 在类被加载的时候执行，且只执行一次；
2. 在同一个类中按照先后顺序依次执行；
3. 在不同类之间，按照类被加载的顺序先后执行；
